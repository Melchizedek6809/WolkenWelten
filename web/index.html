Wolkenwelten Pre-Alpha

Greetings Cybersurfer, you have reached my little corner of the WWW offering a sneak peek of a videogame I have been developing in my spare time.


For now, please do not share this link with others since it is still pretty early in development. Once more gameplay has been added and it seems like there is fun to be had you can go wild. :)


For now only Win/Mac/Lin are natively supported, although there is the wasm version which should work on every OS having a modern Browser. Apart from that, future versions are planned for Android/iOS and the BSDs.

Controls

Mouse / Keyboard

W,S,A,D to move around the world.

Spacebar to jump.

Shift to sneak.

E to shoot/retrieve your grappling hook.

Tab open/close inventory/crafting popup.

LMB to destroy a block.

Mousewheel or 1...8 to select an item..

RMB to use the current item.

Q to drop the current item.


N to start flying around in no-clip mode.


Gamepad

The layout is based on an Xbox360 controller, if you have a different controller it might differ but should be in a similar physical location.

Use the left analog stick to walk around and the right analog stick to look around. Use the D-Pad to select the active item. A for jumping and Y for dropping items. Use the left shoulder trigger for activating the current item and the right one for destroying blocks, the right shoulder button shoots the grappling hook. To open the Inventory press the left Menu button, most probably labeled "Back".


Touchscreen

These controls are really nasty and you should try to get another input device somehow!

Put your thumb on the lower left corner of the screen to move around, in the lower right corner is a zone for looking around. The upper area of the screen is separated into 4 zones used for dropping items, jumping, activating items and destroying blocks. You might wonder how you might select an item, you can't right now ;) Improved touchscreen controls will have to wait until a later time, the current controls were mostly done as a proof of concept.

News / Development history

2019-11-25

Weekly rundown #8

This week was mostly spent fixing bugs, rewriting the code from C++ to C and removing GLM. Although multiplayer was planned for this week I prioritized this switch so I didn't have to port two codebases.


The rewrite surfaced some warts in the codebase which I removed for the most parts, the grapplingHook and character inventory code is strill atrocious for now but will be fixed the next time I add/change something in them.


Apart from that the blockIconGenerator got reworked so it is WebGL compatible enabling working wasm versions again.

Even though it was not intended, the C rewrite and reworking of how memory is allocated seem to have lead to MASSIV wasm performance improvements, especially in Firefox. On a slow day I should investigate why that is.


Keep on keeping on

2019-11-17

Weekly rundown #7

These two weeks I did not have too much time for the game which is why last weeks release was skipped.


Apart from that: OpenGL has been downgraded to 2.1 since there were some platform without support for modern OpenGL which I would like to support (mostly Safari which only supports WebGL 1 and some older iOS/Android devices).

Additionally this week saw a change in Art direction, going for a more minimalist look that should also enable some nice features in the future. Some bugs were fixed and a lot of work has been put into making the codebase more amenable to a Multiplayer Client/Server Architecture.


Ciao.

2019-11-04

Weekly rundown #6

Quite a lot of work has been put into refactoring, especially when it comes to Items which were a mess before, now I have become content with their architecture which should also lay the foundation for Mods.


Other big changes are the Migration from OpenGL 2.1 to OpenGL 3.3 and the usage of offscreen rendering enabling a nicer depth fade and nice visual effects in the future. This also broke a couple of versions which for the most part got fixed, except for wasm which I got to run but sadly at an abyssmal Framerate, hopefully I can fix this in the week to come.

Apart from that, this week other refactoring will get priority, especially the GUI layer and some early tests of LAN Multiplayer which will probably not be stable enough for release by next Sunday, but who knows :)


See you next week, bye!

2019-10-27

Weekly rundown #5

While I originally planned this week to focus on gameplay additions like tools and weapons, I got distracted badly by a rendering bug that once I noticed it was impossible to ignore, in the process of fixing this bug I had to disable the mesh optimizer leading to abysmal performance, to make up for that LOD had to be upped by quite a lot, leading to ugly, blocky islands in the distance. Hopefully I will find a way to reenable the meshoptimizer or envision new ways to render the world in a performant way.

This was quite the setback for me, since I spent quite some time optimizing these routines and have become quite proud of them, still, using them lost me the guarantee of gapfree rendering.


Although I still got to add two Tools, an Axe and a Pickaxe (that can also be used for shoveling dirt) as well as the first real weapon, the Assblaster 9000! Hopefully you enjoy using it and view it as a glimpse of things to come :)


Next week will mostly be spent refactoring the codebase, since the Inventory screen code is a complete mess, Items have pretty much no architecture and there is some cleaning up in the tesselation code due to the now missing meshoptimizer.

Adios, cyber surfer.

2019-10-20

Weekly rundown #4

This week I mostly worked on the build system so I could more easily build redistributable versions and added MacOS and Linux executables. Apart from that the Engine does not depend on legacy OpenGL functionality anymore greatly increasing wasm performance since the compatibility shim has been removed, which also unblocks Mobile/RPi versions.

In addition to this initial support for force-feedback / rumble was added which will probably be toned down a bit in future releases, but for now it seems fun. Continuing from that GamePad controls have been improved since all axis now respect deadzones leading to much better controls on at least an Xbox One controller (probably good for a lot of other controllers as well).

Apart from a lot of technical work the item drop animation has been changed, the crosshair has been changed due to compatibility problems with gles/webgl. The sun got a simple "glow" effect and the block textures were altered a bit and the Grappling Hook should be a lot more reliable now.


That should wrap it up for this week, the coming week I will prioritize Tools and simple Weapons to get some gameplay going.
